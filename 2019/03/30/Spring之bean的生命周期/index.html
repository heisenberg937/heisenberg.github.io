Spring学习之bean的生命周期、依赖注入

Posted on 2019-03-30 Edited on 2023-07-29 In programming Views: Changyan:

最近做的项目要用spring，在这里记点笔记，也是梳理一下重要的知识点。

BeanFactory与ApplicationContext

spring容器是spring框架的核心组件，容器会创建、绑定、配置对象，并且管理他们从创建到销毁整个生命周期。Spring容器使用DI(依赖注入)来管理组成应用的组件，这些对象被称为spring beans。

spring提供了两种不同类型的容器

1，BeanFactory Container

最核心的容器，提供依赖注入支持，由org.springframeword.beans.factory.BearnFactory接口定义。

2，ApplicationContext Container

这个容器添加了更多企业级功能，比如从配置文件中提取文本信息，发布应用事件到相关的监听器，由org.springframework.context.ApplicationContext定义。

一般bean工厂称为IoC容器，ApplicationContext称为应用上下文，有时也称为Spring容器。

对二者用途的划分：BeanFactory是Spring框架的基础设施，面向Spring本身；ApplicationContext面向开发者，应用场合一般直接应用ApplicationContext。

bean的定义

构成用户应用程序主干的对象，他们被Spring的IoC所管理，称为beans.Bean由IoC容器实例化、装配。容器根据配置元数据来创建bean，通过xml或者注解的形式。容器根据配置元数据来决定：如何创建bena，bean的生命周期，bean的依赖。

配置元数据的三种方式：

1.基于XML

2.基于注解

3.基于Java

bean的作用域

当定义一个bean时有选项可以声明作用域。如果想要每次需要时生成一个新的bean，应该将bean的作用域属性声明为prototype；如果想每次需要的时候都返回同一个bean，则应该声明为singleton（默认作用域）。通常，对所有有状态的bean使用原型作用域，对无状态bean使用singleton作用域。其余的作用域还包括request，session，global-session（都与web应用环境相关）。

Bean的后处理器

通过BeanPostProcessor接口定义的方法你可以实现自己的实例化逻辑，依赖解决逻辑等。

你可以配置多个BeanPostProcessors接口，可以通过配置order属性来决定他们的执行顺序。BeanPostProcessors在bean实例对象上起作用，也就是IoC容器先实例化bean实例，然后BeanPostProcessors开始执行。应用上下文(ApplicationContext)自动检测所有实现了BeanPostProcessor接口的bean，并将这些bean注册为后处理器(postprocessors),在容器创建bean之后调用。

bean定义可以包含许多配置信息，包括构造器参数，属性值，构造器特定的信息比如初始化方法、静态工厂方法名等。子类Bean继承父类bean定义的配置信息，子类可以根据需要重写，添加一些值。

第二种注入依赖的方法是属性注入(setter methods)。

1.构造函数注入

当容器调用带参数的类构造器时完成。

2.属性注入

当容器在调用无法构造器或者无参静态工厂方法来实例化bean之后再调用属性设置方法时完成。

建议使用构造函数注入来实现必须的依赖，属性注入来实现可选的依赖。

使用依赖注入之后的代码更整洁，实现了解耦。
